## Library Architecture

### Layered API Structure

```
┌─────────────────────────────────────────────────────────────────┐
│                         APP_API                                  │
│  Application layer - combines service + foundation + web         │
│  simple_app_api                                                  │
├─────────────────────────────────────────────────────────────────┤
│                       SERVICE_API                                │
│  Service layer - JWT, SMTP, SQL, CORS, Rate Limiting,           │
│  Templates, WebSocket, Cache                                     │
│  simple_service_api                                              │
├─────────────────────────────────────────────────────────────────┤
│                     FOUNDATION_API                               │
│  Foundation layer - Base64, Hash, UUID, JSON, CSV,              │
│  Markdown, Validation, Process, Randomizer                       │
│  simple_foundation_api                                           │
└─────────────────────────────────────────────────────────────────┘
```

### By Category

**Foundation Layer** (data encoding, hashing, validation):
- simple_base64 - RFC 4648 Base64 encoding
- simple_hash - MD5, SHA-1, SHA-256, SHA-512 hashing
- simple_uuid - UUID v4 generation
- simple_json - JSON parsing and generation
- simple_csv - CSV parsing and generation
- simple_markdown - Markdown to HTML conversion
- simple_validation - Input validation rules
- simple_process - Process execution helper
- simple_randomizer - Random data generation

**Service Layer** (authentication, database, web services):
- simple_jwt - JWT token creation/verification
- simple_smtp - Email sending
- simple_sql - SQLite database operations
- simple_cors - CORS header handling
- simple_rate_limiter - Request rate limiting
- simple_template - Template rendering
- simple_websocket - WebSocket protocol
- simple_cache - LRU cache with TTL
- simple_logger - Structured logging with JSON

**Web Layer** (HTTP, HTML, frontend):
- simple_web - HTTP server and client
- simple_htmx - Fluent HTML/HTMX builder
- simple_alpine - Alpine.js integration

**API Facades**:
- simple_foundation_api - Unified access to foundation libraries
- simple_service_api - Unified access to service libraries
- simple_app_api - Unified access to all libraries

**Tools & Applications**:
- simple_ci - CI/CD build tool
- simple_showcase - Documentation showcase site
- simple_gui_designer - GUI design tool

## GitHub Pages Setup

To enable documentation for a library:

1. Go to the GitHub repository
2. Settings > Pages
3. Source: Deploy from branch
4. Branch: main, /docs folder
5. Save

The documentation will be available at: `https://ljr1981.github.io/{library-name}/`
